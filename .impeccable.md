## Design Context

### Users
Three user segments, all interacting in a web browser:
- **Developers** registering apps and configuring API keys/permissions for ATProto
- **Community managers** creating and administering communities (members, roles, settings)
- **Individuals** browsing, joining communities, and sharing content

Context is typically daytime desktop use, but mobile access should be fully supported — community managers often check status on the go.

### Brand Personality
**Comfy, collaborative, community.** The interface should feel like a well-organized gathering space — warm and inviting, not corporate or sterile. Think community bookshelf, not SaaS dashboard.

### Aesthetic Direction
- **Tone**: Clean and editorial with warmth. Inspired by Offprint's clarity but with more communal personality.
- **Theme**: Light mode primary, dark mode supported.
- **Palette**: Warm amber/terracotta accent on cream surfaces. Tinted warm neutrals.
- **Anti-references**: Generic admin panels. Cold blue SaaS tools. Cluttered dashboards.

### Design Principles
1. **Warmth over sterility** — Every surface should feel intentional and inviting, never clinical.
2. **Clarity over density** — Show what matters. Use progressive disclosure for complexity (settings, permissions).
3. **Consistency compounds** — Shared patterns across pages build familiarity and trust.
4. **Mobile is real** — Not a shrunken desktop. Adapt layouts meaningfully for touch.
5. **Community first** — The UI should make people and groups feel central, not data.
